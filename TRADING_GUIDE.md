# Детальний гайд: як ефективно торгувати ботом

## Крок 0: Перевірте пари на Bybit

Не всі пари є на Bybit. Перевірте на [Bybit Markets](https://www.bybit.com/trade/usdt):
- ETHUSDT, BTCUSDT — завжди є
- SOLUSDT, XRPUSDT, DOGEUSDT — зазвичай є
- LDOUSDT, інші альткоїни — перевіряйте вручну

Якщо пари немає — Bybit поверне помилку при відкритті позиції.

---

## Крок 1: Налаштуйте пари та суми в config/config.yaml

```yaml
pairs:
  - symbol: SOLUSDT
    timeframe: 1h
    max_usdt: 20      # стартовий капітал $20 на цю пару (тільки перша угода)
    enabled: false
  - symbol: ETHUSDT
    timeframe: 1h
    max_usdt: 30
    enabled: false
```

### Що означають поля

| Поле | Опис |
|------|------|
| `symbol` | Пара на Bybit (формат XXXUSDT) |
| `timeframe` | Таймфрейм (1h, 15m, 4h) |
| `max_usdt` | **Стартовий капітал** у USDT на пару (тільки для першої угоди) |
| `enabled` | Активність через Telegram (▶/⏸) |

### Реінвестування

- **Перша угода**: маржа = `max_usdt` (наприклад, $20).
- **Після закриття з прибутком**: до капіталу пари додається `reinvestment_pct` % прибутку. Наприклад, прибуток $5, реінвест 50% → +$2.50 до капіталу.
- **Наступні угоди**: маржа = поточний капітал пари (без обмежень). Якщо капітал виріс до $22.50 — наступна угода відкриється на $22.50.
- **При збитку**: капітал пари зменшується на суму збитку.
- **Спільний баланс**: всі пари користуються єдиним балансом рахунку. Фактична маржа = min(капітал пари, доступний баланс).

## Крок 2: Налаштуйте стратегію та execution

```yaml
strategy:
  trailing_activation_pct: 0.1   # активувати трейлінг після 0.1% прибутку
  trailing_stop_pct: 0.7         # стоп на 0.7% нижче вершини (long)

execution:
  reinvestment_pct: 50           # % прибутку для реінвестування (1–100). Показується в Налаштування
  leverage: 1                    # плече для всіх пар
  use_limit_orders: true         # лімітні ордери для входу/виходу по сигналу
```

## Крок 3: Деплой змін на Render

1. Змініть `config/config.yaml` локально.
2. Закомітьте та запуште:
   ```bash
   git add config/config.yaml
   git commit -m "Update pairs and max_usdt"
   git push origin main
   ```
3. Render автоматично перезбере сервіс (якщо увімкнено auto-deploy).

## Крок 4: Робота в Telegram

### Старт торговлі

1. **Пари** → список пар з сумами (наприклад, `⏸ SOLUSDT (1h) $20`).
2. Натисніть **▶** біля пари — торговля по ній запускається.
3. Або **Запустити все** — старт по всіх парах з конфігу.

### Що відбувається далі

- Кожні ~60 секунд бот перевіряє сигнали для кожної активної пари.
- Якщо є сигнал Long/Short і позиції немає — відкривається позиція на виділений капітал пари (перша угода = max_usdt, далі — з реінвестом).
- Якщо є сигнал закриття або трейлінг-стоп — позиція закривається, прибуток частково реінвестується.

### Зупинка

- **⏸** біля пари — зупинка тільки по цій парі (відкриту позицію не закриває, лише нові не відкриває).
- **Зупинити все** — зупинка всіх пар.

## Типовий workflow

1. Поповнити Unified-акаунт на Bybit (мінімум ~$100 для 4 пар по $20–30).
2. Переконатися, що API-ключі мають права на торговлю.
3. У конфігу вказати пари з `max_usdt`.
4. Запушити зміни, дочекатися деплою на Render.
5. У Telegram: **Запустити все** або запустити окремі пари.
6. Стежити за сповіщеннями та балансом.

---

## Важливі нюанси

1. **Одна позиція на пару** — бот не відкриває кілька позицій по одній парі.
2. **Реінвест** — `reinvestment_pct` (Налаштування) задає % прибутку, що додається до капіталу пари. Обмежень на розмір подальших угод немає.
3. **Трейлінг-стоп завжди ринковий** — для швидкого закриття при просадці.
4. **Лімітні входи** — можуть не виконатися, якщо ціна пішла далі; для швидких ринків можна встановити `use_limit_orders: false`.
5. **Збереження капіталу** — дані зберігаються в `data/pair_capital.json`. Локально переживають перезапуск; на Render файлова система ефемерна — при деплої капітал скидається до max_usdt.
6. **Testnet** — для тестів встановіть `BYBIT_TESTNET=true` в Render Environment.
